concept

orbit is a four-track euclidean drum machine for norns. Each of its four tracks — kick, clap, closed hat, open hat — rides its own concentric ring on screen, and the four rings turn together on a single 1/16-note clock. A playhead sweeps each ring; when it crosses an active step, that track fires. The visual and the sequencer are the same object: the rings you see are the pattern you hear.

The whole instrument — sequencer, mixer, delay, and visualizer — is folded into that single turning-rings screen. There is no separate pattern view; muting a track simply dims its ring, and the selected track's ring is highlighted so you know which one the encoders will edit.

the euclidean heart

Every track's pattern is generated, not drawn by hand. A Euclidean algorithm spreads K beats as evenly as possible across N steps — the maximally-even distribution that underlies so much world percussion. You set, per track, the number of beats and a rotation that shifts where the pattern starts.

The implementation is a compact accumulator rather than a recursive Bjorklund: it walks the steps and drops a beat whenever a running fractional counter rolls over (bucket += K; if bucket >= N then bucket -= N; place beat). Same even spread, and trivially rotatable by index-shifting the resulting boolean table. (orbit fixes N at 16 steps.)

four tracks

orbit uses Ack, the norns sample-playback engine — not a custom engine of its own. It drives four Ack voices, one per track, controlling each voice's volume, pitch, and low-/high-pass cutoff from the script. The defaults below ship a working beat the moment you load it.

per-track delay — the part that makes it orbit

A standard drum machine sends everything to one delay bus. orbit instead gives each track its own independent delay, and the delays are trigger-based: when a track fires, it schedules its own ghost retriggers directly — each a decaying echo at a tempo-synced division. This is the feature the whole instrument is built around.

Each track's delay has two controls: a decay (how much quieter each successive echo is, in dB) and a division (the tempo-synced spacing of the echoes). Because each track delays independently, you can run a slow dotted tail on the kick while the hats stutter fast — polyrhythmic echo textures a single shared delay bus can't make.

devlog

2026-04-01 — initial implementation

orbit lands whole. Four euclidean tracks on the Ack engine, each a concentric ring on a single 1/16 clock, with the accumulator-based euclidean generator, the per-track trigger-based ghost-retrigger delay (decay in dB + tempo-synced division), a K2/K3 hold-modifier control scheme, and the rings-as-sequencer screen. Default kit: kick E(4,16), clap E(2,16) rotated, closed hat E(8,16), open hat E(3,16) rotated.
